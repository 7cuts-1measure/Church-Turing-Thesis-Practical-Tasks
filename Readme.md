# Оптимизация программы

## Условие

Дан текст программы на игрушечном языке программирования [toy_lang.md](./toy_lang.md).

Необходимо:

1. распарсить программу;
2. построить для неё CFG в канонической форме;
3. выполнить статический анализ программы:

   * распространение констант (SCCP),
   * анализ диапазонов значений (range analysis);
4. для каждого оператора `print` определить, можно ли **заранее (до выполнения программы)** однозначно определить его результат.

Язык описан в файле: [toy_lang.md](./toy_lang.md)

## Что требуется соптимизировать

Для каждого оператора:

```c
print(boolean);
```

необходимо определить, принимает ли выражение `boolean`:

* всегда значение `true`,
* всегда значение `false`,
* или его значение зависит от входных данных / пути выполнения.

И кроме того необходимо узнать, доходит ли поток исполнения до print, хотя бы потенциально.


## Формат вывода

Для каждого оператора `print`, который **может потенциально выполниться** (в порядке их появления в программе) необходимо вывести **одну строку**:

```text
true
false
?
```

Файл с результатом записывается в `<input_path>.ans`

## Значения

* `true` — выражение **всегда истинно**, независимо от:

  * входных данных (`read()`),
  * ветвлений программы;

* `false` — выражение **всегда ложно**, независимо от входных данных и ветвлений;

* `?` — значение:

  * зависит от `read()`, или
  * зависит от пути выполнения, или
  * не может быть точно определено.


## Важные замечания

* учитывать нужно **все возможные пути выполнения**;
* `read()` возвращает произвольное значение;
* переменные изначально имеют значение `undef`;
* анализ должен быть **консервативным**:
  * если нельзя доказать `true` или `false`, нужно выводить `?`.
  * если нельзя доказать, что инструкция недостижима, значит она потенциально достижима

## Пример

### Вход

```c
x = 5;
if (x < 10) {
    y = 10;
} else {
    y = 3;
}
print(y > 5);
```

### Выход

```text
true
```

## Краткая формулировка

Для каждого потенциально достижимого оператора `print` необходимо определить, можно ли заранее доказать, что он всегда печатает `true`, всегда `false`, или его значение зависит от выполнения программы (`?`).
